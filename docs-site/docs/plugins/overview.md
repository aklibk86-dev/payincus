# 插件中心

PayIncus 插件中心用于安装和管理第三方插件。插件可以提供后台配置页，也可以通过受控客户端扩展点向用户端增加入口或页面。

插件不会直接修改 PayIncus 源码。安装后的文件保存在运行态目录：

```text
/opt/incudal/plugins
/opt/incudal/plugin-data
/opt/incudal/plugin-logs
/opt/incudal/plugin-staging
```

这些目录会在 OTA 更新和回滚时保留。

## 能力

- 上传 `.tar.gz` 插件包安装
- 从 GitHub 插件市场安装
- 启用、禁用和卸载插件
- 查看安装任务日志
- 安装后在后台左侧菜单显示插件设置入口
- 通过独立插件设置页管理业务配置
- 通过 sandbox iframe 加载第三方后台或用户端插件页面
- 使用内置模板开发插件，例如 `plugin-templates/ai-ticket-agent-plugin` 提供 AI 工单草稿插件模板

## 后台界面

插件中心在后台内部分为三个页面：

- 已安装：上传插件包、查看插件列表、启用/禁用/卸载插件和打开插件设置。
- 插件市场：单独展示 GitHub 市场索引，管理员可以刷新市场并安装市场插件。
- 安装任务：查看上传安装、市场安装、启用、禁用和卸载任务；任务列表固定每页最多 7 条，更多任务通过分页查看，右侧日志面板显示所选任务输出。

插件中心只负责安装、启用、禁用、卸载和查看任务。插件安装后，如果 manifest 声明了 `admin.plugins.settings` 后台设置页，PayIncus 会把插件设置入口加入后台左侧菜单；未启用插件也可以先进入设置页维护配置，页面会提示当前启用状态。设置页面使用独立路由打开，不再放在插件中心右侧详情栏里。

## AI 工单插件模板

`plugin-templates/ai-ticket-agent-plugin` 是官方插件模板，用于第三方开发 AI 工单辅助插件。模板要求 PayIncus `v0.3.2` 或更高版本。

官方市场会提供 `AI 工单助手` 插件包。管理员需要先在生产环境配置 `PLUGIN_MARKET_INDEX_URL`，然后进入后台插件中心的“插件市场”页刷新并安装。安装完成后，后台左侧菜单会显示 `AI 工单助手` 设置入口；需要实际生成草稿或接管回复时，再在“已安装”页启用插件。

`AI 工单助手` 使用独立中文设置页，不需要编辑配置 JSON。设置页可以维护启用开关、接管模式、OpenAI 兼容模型地址、模型名、API key、温度、超时、自动回复分类、置信度阈值、每日上限、单工单上限、冷却时间、AI 身份标识和自定义系统提示词。API key 保存后会加密存储，页面不会回显；留空保存时会保留已有密钥。

启用后，后台工单详情页可以生成 AI 回复草稿。草稿只会填入回复框，不会自动发送；管理员必须人工确认后再提交。

插件也提供受控接管回复接口。该接口会重新生成回复、执行安全检查、执行敏感工单强制人工规则，并按插件配置限制每日发送量、单工单发送量和冷却时间。它只会写入一条客服侧消息，不会修改工单状态。

当插件配置为 `auto` 模式时，服务端调度器每 2 分钟扫描一次官方/系统工单。它只处理最新消息来自用户、非紧急、非 abuse、分类在 `autoReplyCategories` 内且未触发人工接管规则的工单；第三方托管节点工单不会被自动接管。

模板依赖两个受控接口：

```text
POST /api/tickets/:id/ai/context
POST /api/tickets/:id/ai/draft
POST /api/tickets/:id/ai/reply
```

这些接口只允许管理员访问，必须启用 `com.payincus.ai-ticket-agent` 插件并授予对应权限。上下文只读取当前工单所属用户的脱敏摘要，不返回管理员内部备注、支付回调、密钥、root 密码或其他用户数据。

插件设置页还会读取一个安全状态接口：

```text
GET /api/tickets/ai/status
```

该接口只返回插件是否启用、当前模式、模型是否已配置、权限状态、自动分类、置信度阈值和限流配置等运营摘要，不返回模型地址、API key、后台路径、原始工单内容或用户数据。

## 安全边界

- 只有超级管理员可以安装、启用、禁用和卸载插件。
- 用户端插件不能访问 `/api/admin/*`。
- 插件包不能包含绝对路径、`../` 路径、软链接或硬链接。
- 市场插件必须来自 GitHub Release artifact，并校验 SHA256。
- 当前版本不执行插件自带 shell 脚本，也不加载任意后端代码。
- AI 工单接管回复必须通过独立 `ticket:ai:reply` 权限，并会对退款、争议、账号安全、风控、删除/重装/迁移实例、凭据或后台细节、交付异常强制转人工。
- AI 工单接口不能调账、删除实例、修改工单状态或执行其他资源操作。
- AI 自动回复只在 `auto` 模式运行，并受每日上限、单工单上限和冷却时间约束。
