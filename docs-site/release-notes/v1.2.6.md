# v1.2.6

## 修复

- 修复交易所后台余额冻结、解冻、人工调整的原子性问题：余额变更、交易所钱包流水和交易所审计日志现在会在同一个数据库事务内提交。
- 如果审计日志写入失败，本次后台余额操作会整体回滚，避免出现“资金已变化但接口返回失败，管理员重试后重复冻结或重复调整”的风险。
- 补强交易所后台资金操作守卫，确保后续不会把钱包资金变更和审计日志重新拆成两段事务。
